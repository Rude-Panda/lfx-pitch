08 — The Solution

Three Things
Nobody Else Has Combined.

1 — Agent-First, Not Tool-First

Claude with extended thinking — reasons before acting, plans multi-step workflows, not just single-turn prompts. Uses BM25 semantic search to find the right model across 50+ tools without loading them all into context.

60–80% lower cost per request via Anthropic prompt caching (5-min TTL). Up to 20K thinking tokens per request.

2 — Spatial Canvas

Custom-built infinite canvas — not off-the-shelf. 9 element types. Assets appear as they generate, mid-stream. Mind maps, freehand drawing, groups, z-ordering. The canvas is the creative memory.

Closest analogy: Figma's canvas — but for AI generation across every media type.

3 — Model-Agnostic (50+ Models)

Not locked to any single provider. Veo 3.1, Sora 2, Kling 3.0, ElevenLabs, Meshy, and 45+ more. Adding a new model requires ~20 lines of config. If a better model launches tomorrow, it's integrated within minutes.

Decisions That Can't Be Copied — Only Earned

These are non-obvious choices that emerged from building, shipping, and debugging — not from reading documentation.

DecisionWhy It Matters
BM25 over vector search for tool discovery Zero cold-start, no embeddings infra, exact tag matching across 52 models
Fire-and-forget generation (return job_id immediately) Generation takes 30–180s. Synchronous waiting kills UX. Discovered from watching it fail.
Credit deduction before generation, refund on failure Prevents balance drift when concurrent requests race. Atomic via Supabase RPC.
All tools deferred — none loaded upfront Loading 52+ tool schemas per request would exhaust context. BM25 lazy-loads only what's needed.
SSE primary, WebSocket feature-flagged SSE works through all reverse proxies without config. WS requires Redis — made optional.
Thinking blocks stripped from prior iterations Extended thinking blocks are large. Keeping them across 15 iterations would overflow context.
Prompt cache boundary at older messages, not latest Cache hit requires exact prefix match. Boundary must stay stable as conversation grows.
Custom canvas over React Flow 9 element types, sub-pixel hit-testing, R-tree spatial index. Any library update would break assumptions.
